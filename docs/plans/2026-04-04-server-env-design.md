# 服务端环境页重构设计

## 背景

当前“高级工具 -> 服务端环境”页面虽然功能完整，但结构仍然接近配置清单直出：

- 管理员模式、系统动作、服务通知、分类变量全部连续堆叠
- 每个环境变量同时展示键名、说明、类型、值预览、选项、编辑、删除
- 分类一多时页面纵向过长，移动端阅读路径不清晰
- 高频操作与低频信息没有拉开层级，首屏噪音过大

这导致页面更像“后端配置 dump”，不像手机端维护工具页。

## 目标

把该页重构为面向移动端的“分类切换式工作台”，保留全部能力，但显著提升：

- 首屏清晰度
- 分类浏览效率
- 单项变量可读性
- 编辑操作聚焦度
- 与首页 / 核心 / 日志页的一致性

## 设计原则

1. 功能不减，但默认信息量下降
2. 首屏只放主路径，不平铺全部细节
3. 变量项默认摘要展示，详细信息按需展开
4. 危险操作降噪，不在每一行里常驻抢眼
5. 视觉语言与现有沉浸式设置体系保持一致

## 页面结构

页面采用四段结构：

### 1. 沉浸头部

- 标题：`服务端环境`
- 副标题：强调“管理员模式与环境变量维护”
- 保留返回按钮

头部只承担定位作用，不再承担说明文档作用。

### 2. 管理员状态卡

卡片顶部显示当前模式：

- 普通模式
- 管理员模式

卡片中部保留两个紧凑状态项：

- 环境内置 Token
- 当前会话权限

卡片底部保留会话 Token 输入与两个动作：

- 验证并启用
- 退出管理员模式

反馈消息改为单行状态提示，不单独占据大型区域。

### 3. 系统动作卡

集中承载高风险或运维动作：

- 刷新配置
- 清理缓存
- 重新部署

卡片底部仅保留少量关键信息：

- 服务版本
- 仓库
- 分类数

如接口返回 `notice`，在此区域以简洁提示块单独展示，不参与变量列表排版。

### 4. 环境变量工作台

这是页面主体，改为两层结构：

#### 第一层：分类切换

使用与记录页一致的胶囊式切换器，一次只展示一个分类。

每个分类标签展示：

- 分类名
- 数量摘要

分类顺序优先稳定，避免每次刷新后跳动。

#### 第二层：变量列表

当前分类下的变量以紧凑行卡片展示，每项默认只显示：

- 变量名
- 类型标签（仅在非普通 text 时显示）
- 当前值摘要
- 一句简短说明

操作入口不再使用常驻“编辑 / 删除”文字按钮，改为右上角 `更多` 或次级操作入口。

## 变量项展开规则

环境变量列表采用“默认收起，按需展开”：

### 收起态

- 一屏可看到更多变量
- `options` 不再全量平铺
- 值只显示摘要，不占大面积高度

### 展开态

展开后显示：

- 完整值
- 可选值 chips
- 编辑入口
- 删除入口

展开区仍保持紧凑，避免出现多层嵌套卡片。

## 视觉方向

沿用当前管理器重构版的设计语言：

- 沉浸背景
- 低噪音面板
- 圆角但不过分厚重
- 高信息密度
- 标签数量受控

重点通过以下方式建立层级：

- 标题字号
- 卡片留白
- 次级文案弱化
- 状态色仅用于关键状态

不再依赖大量标签、重复按钮或多层边框。

## 交互约束

- 不新增后端接口
- 继续使用当前 `serverConfig` 数据结构
- 继续使用现有编辑 / 删除弹窗逻辑
- 管理员 Token 仍只保存在当前会话
- 页面加载、错误、空态逻辑保持完整

## 验收标准

重构完成后应满足：

1. 首屏不再出现多个大卡片连续堆叠变量内容
2. 环境变量只展示当前分类，不全量展开所有分类
3. 单个变量默认只显示摘要信息
4. 编辑与删除不再在每一行高噪声常驻
5. 页面整体风格与首页、核心页、日志页一致
6. 现有管理员模式、刷新配置、清理缓存、重新部署、编辑变量、删除变量能力全部可用
