# 核心页重构设计

## 目标

把“核心”页从偏展示型大卡片改成更接近方案 B 的列表控制台：

- 当前核心保留，但压缩成低矮状态条
- 搜索、筛选、核心操作前置且集中
- 已安装核心列表成为页面主角
- 整体卡片更扁平、更紧凑，减少不必要装饰

## 设计结论

采用方案 B：列表控制台版。

### 结构顺序

1. 顶部标题栏
   - 标题保留“核心”
   - 副标题显示当前核心与已安装数量
   - 右侧状态 pill 显示“x 个可更新”或“已对齐”

2. 当前核心状态条
   - 不再使用高 Hero 卡
   - 仅保留当前核心名称、分支 / 版本 / 提交摘要
   - 提供两个高频动作：查看详情、安装更新 / 重新安装
   - 无当前核心时改为引导安装

3. 管理工具区
   - 搜索框
   - 过滤器：全部 / 当前 / 可更新
   - 操作按钮：检查全部更新、安装新核心

4. 已安装核心列表
   - 作为页面主体
   - 当前核心置顶
   - 可更新核心优先
   - 条目高度收紧，状态标签缩小
   - 点击条目仍进入详情页

## 视觉原则

- 保持浅色默认主题，不改全局色板
- 卡片圆角与阴影收敛，减少厚重感
- 行高、标题字号回归真实移动端尺寸
- 用分区与列表秩序体现“高级控制台”，而不是靠大面积装饰

## 实现边界

- 不新增后端能力
- 不改详情页信息结构与危险操作位置
- 只重构 `CoreHubScreen.kt` 主页面相关布局
- 允许抽出轻量级展示文案 / 状态格式化 helper，便于测试

## 验收标准

1. 当前核心不再是大 Hero 卡，而是紧凑状态条
2. 搜索、筛选、安装 / 更新操作集中展示
3. 已安装核心列表视觉上成为页面主体
4. 页面整体比旧版更紧凑、更扁平
5. 功能不丢失：查看详情、检查更新、安装新核心、筛选、搜索都可用
